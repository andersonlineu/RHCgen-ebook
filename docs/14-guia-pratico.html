14  Guia Prático de Instalação e Uso

Este guia tem como objetivo orientar, de forma simples, a instalação e o uso inicial do pacote RHCgen. O pacote foi desenvolvido para automatizar o processamento dos arquivos do Registro Hospitalar de Câncer (RHC), disponibilizados pelo IntegradorRHC no formato .dbf.

O RHCgen permite transformar vários arquivos brutos do RHC em uma base única, padronizada, recodificada e pronta para análises epidemiológicas, estudos de qualidade dos dados e produção de indicadores.

14.1 Pré-requisitos

Antes de usar o RHCgen, é necessário ter dois programas instalados:

  • R → cran.r-project.org
  • RStudio → posit.co/download/rstudio-desktop

O R é a linguagem utilizada para executar o pacote. O RStudio é o programa que facilita o uso do R, permitindo escrever comandos, visualizar resultados e organizar arquivos.

Dica

Recomenda-se instalar primeiro o R e depois o RStudio.

14.2 Onde obter os arquivos do RHC

Os arquivos utilizados pelo RHCgen devem ser baixados no IntegradorRHC, disponível em:

https://irhc.inca.gov.br/RHCNet/

Os dados são disponibilizados em arquivos no formato .dbf, com nomes iniciados por rhc, como:

rhc00.dbf   rhc10.dbf   rhc20.dbf
rhc01.dbf   rhc11.dbf   rhc21.dbf
rhc02.dbf   rhc12.dbf   rhc22.dbf
                        rhc23.dbf
ImportanteMantenha os nomes originais dos arquivos

O RHCgen utiliza o nome do arquivo para identificar automaticamente o ano do banco e criar a variável Ano_do_Banco. Não renomeie os arquivos baixados do IntegradorRHC.

14.3 Como organizar os arquivos

Antes de usar o pacote, crie uma pasta exclusiva no computador para armazenar os arquivos do RHC.

Exemplo de organização:

C:/RHCgen/dados_rhc/rhc00.dbf
C:/RHCgen/dados_rhc/rhc01.dbf
C:/RHCgen/dados_rhc/rhc02.dbf
C:/RHCgen/dados_rhc/rhc03.dbf

Todos os arquivos que serão processados devem estar na mesma pasta.

14.4 Como abrir o RStudio

Após instalar o R e o RStudio, abra o programa RStudio. A tela é geralmente dividida em quatro áreas:

Área Função
Console Local onde os comandos são executados
Script Local onde os comandos podem ser escritos e salvos
Environment Local onde aparecem os objetos criados
Files/Plots/Packages Local onde aparecem arquivos, gráficos e pacotes

Para criar um novo script: File → New File → R Script

14.5 Como definir a pasta de trabalho

Antes de executar o RHCgen, o R precisa saber em qual pasta estão os arquivos .dbf. Use o comando setwd():

setwd("C:/RHCgen/dados_rhc/")
Nota

O caminho da pasta deve ser adaptado ao local onde os arquivos foram salvos no seu computador. No Windows, use barras / ou barras duplas \\.

Para verificar se o R está usando a pasta correta:

getwd()        # mostra a pasta atual
list.files()   # lista os arquivos disponíveis

Se os arquivos .dbf aparecerem na lista, a pasta foi definida corretamente.

14.6 Como instalar o RHCgen

O RHCgen está disponível no GitHub. Para instalá-lo, primeiro instale o pacote remotes:

install.packages("remotes")

Depois, instale o RHCgen:

remotes::install_github("andersonlineu/RHCgen")
Nota

A instalação pode levar alguns minutos, especialmente se o R precisar instalar pacotes auxiliares.

14.7 Como carregar o pacote

Após a instalação, carregue o pacote para uso:

library(RHCgen)

Este comando deve ser executado sempre que o RStudio for aberto novamente e o usuário quiser utilizar o pacote.

14.8 Como processar os dados automaticamente

A forma mais simples de usar o RHCgen é por meio da função principal construir_banco(), que executa automaticamente todas as etapas do processamento:

data_rhc_final <- construir_banco()

A função executa automaticamente:

  • Leitura dos arquivos .dbf
  • Junção dos arquivos em um banco único
  • Criação da variável Ano_do_Banco
  • Padronização dos nomes das variáveis
  • Conversão dos tipos das variáveis
  • Recodificação das categorias
  • Harmonização de estados e municípios
  • Padronização de códigos clínicos e hospitalares
  • Organização das colunas
  • Avaliação da qualidade dos dados

14.9 Como verificar o banco criado

Após executar a função, o banco aparecerá na aba Environment do RStudio. Também é possível verificar com os comandos:

dim(data_rhc_final)    # número de linhas e colunas
head(data_rhc_final)   # primeiros registros
names(data_rhc_final)  # nomes das variáveis
str(data_rhc_final)    # estrutura das variáveis

14.10 Como filtrar o banco

Após construir o banco completo, use a função filtrar_banco() para selecionar apenas os registros de interesse. A estrutura geral é:

dados_filtrados <- filtrar_banco(
  dados = data_rhc_final,
  argumento = valor
)

14.10.1 Filtros disponíveis

Argumento Descrição Exemplo
cid3digitos CID-O com 3 dígitos "C44"
cid4digitos CID-O com 4 dígitos "C44.0"
ano_inicio_triagem / ano_fim_triagem Período de triagem 2018, 2020
ano_inicio_diagnostico / ano_fim_diagnostico Período de diagnóstico 2015, 2019
ano_inicio_primeira_consulta / ano_fim_primeira_consulta Período da 1ª consulta 2016, 2020
ano_inicio_tratamento / ano_fim_tratamento Período de início do tratamento 2014, 2019
idade_inicio / idade_fim Faixa etária 18, 80
tipo_de_caso Tipo de caso "Analítico"
sexo Sexo "Feminino"
estado_residencia UF de residência "RJ"
uf_unidade_hospital UF do hospital "RJ"
primeiro_tratamento_hospital Primeiro tratamento "Cirurgia"
origem_do_encaminhamento Origem "SUS"
ano_do_banco_inicio / ano_do_banco_fim Ano do banco 2014, 2019

14.10.2 Exemplos de uso

Filtrar por tipo de câncer:

banco_pele <- filtrar_banco(dados = data_rhc_final, cid3digitos = "C44")
banco_mama <- filtrar_banco(dados = data_rhc_final, cid3digitos = "C50")
banco_prostata <- filtrar_banco(dados = data_rhc_final, cid3digitos = "C61")

Filtrar por estado de residência:

dados_rj <- filtrar_banco(dados = data_rhc_final, estado_residencia = "RJ")

# Mais de um estado:
dados_sudeste <- filtrar_banco(
  dados = data_rhc_final,
  estado_residencia = c("RJ", "SP", "MG", "ES")
)

Múltiplos filtros combinados:

dados_filtrados <- filtrar_banco(
  dados              = data_rhc_final,
  cid3digitos        = "C44",
  ano_inicio_triagem = 2018,
  ano_fim_triagem    = 2020,
  idade_inicio       = 20,
  idade_fim          = 80,
  tipo_de_caso       = "Analítico",
  sexo               = NULL,        # NULL = sem filtro por sexo
  estado_residencia  = c("RJ", "MG"),
  origem_do_encaminhamento = "SUS"
)
NotaA função não altera o banco original

filtrar_banco() cria um novo dataframe com os registros selecionados. O banco original data_rhc_final permanece intacto.

14.11 Como salvar o banco processado

A função construir_banco() cria o banco no ambiente do R, mas não salva automaticamente o arquivo no computador.

Formato RDS (recomendado para reutilização no R):

saveRDS(data_rhc_final, "data_rhc_final.rds")

# Para abrir posteriormente:
data_rhc_final <- readRDS("data_rhc_final.rds")

Formato CSV (compatível com Excel, LibreOffice e outros softwares):

write.csv(data_rhc_final, "data_rhc_final.csv", row.names = FALSE)
Dica

Em bases muito grandes, o arquivo .csv pode ficar pesado. Para análises no R, o formato .rds costuma ser mais eficiente.

14.12 Como executar as etapas separadamente

Embora o uso mais simples seja pela função construir_banco(), também é possível executar as etapas uma a uma:

data <- lerarquivoDBF()
data <- renomear_colunas(data)
data <- modificar_tipo_variavel(data)
data <- recodificar_variaveis(data)
data <- renomear_siglas_estados(data)
data <- renomear_codigo_municipio_residencia(data)
data <- renomear_codigo_municipio_hospital(data)
data <- renomear_CID_3digitos(data)
data <- renomear_tipo_histologico(data)
data <- renomear_estadiamento_clinico(data)
data <- classificacao_CID_0(data)
data <- renomear_clinica(data)
data <- renomear_CNES(data)
data <- ordenando_colunas(data)

Esse modo é útil quando o usuário deseja acompanhar cada etapa do processamento separadamente ou aplicar apenas parte do pipeline.

14.13 Como avaliar a qualidade dos dados

Após a construção do banco, execute a análise de qualidade:

analise_qualidade_dos_dados(data_rhc_final)

Essa função realiza duas avaliações principais:

  • Completude — identifica variáveis com maior proporção de dados ausentes ou preenchidos como “Sem informação”
  • Consistência — identifica códigos inválidos, idades inconsistentes, inconsistências entre datas clínicas e incompatibilidades entre sexo e localização tumoral

Quando possível, a função também gera um relatório em HTML com os resultados da auditoria, aberto automaticamente no navegador.

14.14 Como avaliar a completude por ano

tabela_completude_ano <- analise_completude_ano(data_rhc_final)

Essa função retorna uma tabela com as variáveis nas linhas e os anos nas colunas. Cada célula mostra o percentual de completude e uma classificação abreviada:

Percentual Classificação Sigla
≥ 95% Excelente E
90% a < 95% Bom B
80% a < 90% Regular R
50% a < 80% Ruim RU
< 50% Muito ruim MR

Exemplo de resultado: 95.2% (E) · 78.4% (RU) · 42.1% (MR)

14.15 Exemplo completo

# 1. Definir a pasta onde estão os arquivos DBF
setwd("C:/RHCgen/dados_rhc/")

# 2. Instalar o pacote remotes (apenas na primeira vez)
install.packages("remotes")

# 3. Instalar o RHCgen diretamente do GitHub (apenas na primeira vez)
remotes::install_github("andersonlineu/RHCgen")

# 4. Carregar o pacote
library(RHCgen)

# 5. Construir o banco final automaticamente
data_rhc_final <- construir_banco()

# 6. Verificar o tamanho do banco
dim(data_rhc_final)

# 7. Visualizar as primeiras linhas
head(data_rhc_final)

# 8. Avaliar a qualidade dos dados
analise_qualidade_dos_dados(data_rhc_final)

# 9. Avaliar a completude por ano
tabela_completude_ano <- analise_completude_ano(data_rhc_final)

# 10. Salvar o banco final em formato RDS
saveRDS(data_rhc_final, "data_rhc_final.rds")
